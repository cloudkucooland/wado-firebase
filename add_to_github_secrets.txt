### GitHub Actions Secrets Guide for WADO

To activate the CI/CD pipeline, add the following secrets to your GitHub repository (Settings > Secrets and variables > Actions):

1. GCP_SA_KEY
   - Value: The full JSON key for a Service Account.
   - Required Permissions: "Cloud Functions Developer", "Service Account User", and "Artifact Registry Administrator".
   - How to get it:
     a. Go to GCP Console > IAM & Admin > Service Accounts.
     b. Select or create a service account (e.g., the one in your deploy scripts).
     c. Go to the "Keys" tab > Add Key > Create new key > JSON.
     d. Paste the entire content of that file here.

2. FIREBASE_SERVICE_ACCOUNT_OSL_DAILYOFFICE
   - Value: The Service Account key specifically for Firebase Hosting.
   - How to get it:
     Run this command on your local machine:
     `firebase init github`
     It will guide you through the process and automatically add the secret to your GitHub repo if you have the rights, or provide the value to you.

3. GITHUB_TOKEN (Automatic)
   - You do NOT need to add this manually. GitHub provides this automatically to the workflow.

---
Note: Never commit the actual values of these secrets to the repository.
